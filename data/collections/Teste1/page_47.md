# Porting Extension Modules to Python 3 [¶](https://docs.python.org/3/howto/cporting.html\#porting-extension-modules-to-python-3 "Link to this heading")

We recommend the following resources for porting extension modules to Python 3:

- The [Migrating C extensions](http://python3porting.com/cextensions.html) chapter from
_Supporting Python 3: An in-depth guide_, a book on moving from Python 2
to Python 3 in general, guides the reader through porting an extension
module.

- The [Porting guide](https://py3c.readthedocs.io/en/latest/guide.html) from the _py3c_ project provides opinionated
suggestions with supporting code.

- [Recommended third party tools](https://docs.python.org/3/c-api/intro.html#c-api-tools) offer abstractions over
the Python’s C API.
Extensions generally need to be re-written to use one of them,
but the library then handles differences between various Python
versions and implementations.
